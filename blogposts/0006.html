Head First C

22 August 2012

I've used the C programming language throughout my degree and on various projects, and there is no doubt it is an amazing tool. It gives you complete dominion over your computer, and when you start talking about microprocessors and Arduinos, there's really no other option.

However, it's tricky...really tricky. You're down to the nuts and bolts of your computer, which can be both a blessing and a curse. Suddenly you have to worry about memory, file descriptors and pipes. And if, like me you're experience of programming has been built up from high level languages like Javascript and Python, you probably don't have a clue what's going on, and it's scary.

In the past I've just blindly gone at it, adding *s and &s all over the place until it works. But now I decided to finally get over my fear and learn C properly.

I bought the book "Head First C" after a recommendation from Hacker News. It's readable, witty, and I now feel like a C king. What's more, it actually goes in to a lot of practical detail about computer science and operating systems that you don't get told anywhere else (for us non computer scientists). So many little things were cleared up - for example, how compilers work, what pipes are, memory management, dynamic libraries.

I've never thought that a book can solve anything when it comes to coding - so many of the books I buy end up unread and useless compared to online resources. However, Head First C gave me the focus to actually learn this damned language! Highly recommended.
