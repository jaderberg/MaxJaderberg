Adding Your .bash_profile To Matlab's Bash Environment

28 November 2012

When calling system() in Matlab, you probably want all your aliases and environment variables available to Matlab. If you didn't know, system() allows you to call shell commands from Matlab code as if you were in a terminal - e.g. s = evalc('system(''ls -l | wc -l'')') will return a string of the number of files in the current directory.

However, Matlab has its own bash profile so anything you have defined in your ~/.bash_profile (or similar) file will not work when calling system() in Matlab.

The solution is to import your current bash profile in to Matlab by adding the following line to your startup.m file:


setenv('BASH_ENV','~/.bash_profile');

You can edit your startup file by typing edit startup at the Matlab prompt. And that's it - now you have all your environment variables and definitions from your bash profile available to Matlab.
